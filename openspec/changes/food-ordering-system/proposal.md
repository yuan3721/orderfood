## Why

公司需要一套订餐系统来解决员工/用户每日点餐的需求。目前点餐流程可能依赖人工统计，效率低且容易出错。通过数字化的订餐系统，可以实现菜品管理、订单管理、在线支付等功能，提升用户体验和商家运营效率。

## What Changes

- 新增 C 端微信小程序：用户可以浏览今日菜单、选择菜品、提交订单、在线支付、查看历史订单
- 新增 B 端微信小程序：商家可以管理菜谱、查看订单、推送通知、导出订单数据
- 新增服务端 API：提供用户认证、菜单管理、订单管理、支付回调等核心接口
- 新增数据库：存储用户、菜谱、订单、订单明细等数据

## Capabilities

### New Capabilities

- `user-mini-program`: C 端微信小程序，面向普通用户，提供菜品浏览、下单、支付、订单历史等功能
- `merchant-mini-program`: B 端微信小程序，面向商家，提供菜谱管理、订单管理、通知推送等功能
- `backend-api`: 服务端 RESTful API，提供用户认证、菜单、订单、支付等接口
- `database-schema`: 数据库设计，包含用户表、菜谱表、订单表、订单详情表、商家表

### Modified Capabilities

（无现有能力需要修改）

## Impact

- **代码**: 创建三个独立项目 - C端小程序、B端小程序、服务端
- **API**: 新增约15个 RESTful 接口
- **依赖**: 
  - 微信小程序 SDK
  - 微信支付 SDK
  - Node.js/Koa 或 Python FastAPI
  - MySQL/PostgreSQL 数据库
  - 对象存储（OSS/COS）用于图片存储
- **系统**: 需要部署服务端、配置微信小程序、配置微信支付商户
